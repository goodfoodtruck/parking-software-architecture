# 004. Technologies utilisées pour le backend
## Contexte:
Plusieurs options sont envisageables pour le développement du backend 
qui hébergera la logique métier: Java, Rust, NodeJS
L'équipe de développement est à l'aise avec NodeJS.



## Décision:
Dans ce contexte, utiliser un langage de programmation orienté performances
n'est pas pertinent. 
Un serveur développé en NodeJS avec Typescript sera capable de 
répondre au besoin, d'autant plus que l'équipe connait bien cette technologie.



## Conséquences:
#### Positives (+) :
positive: le développement des features sera plus rapide et le code 
suivra les bonnes pratiques de développement avec cette technologie. Typescript permettra à l'équipe
de rendre le code de la logique métier plus robuste, moins sensible aux bugs.
